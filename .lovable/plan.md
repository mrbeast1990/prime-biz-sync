# خطة إضافة نظام تسجيل الدخول وإدارة المستخدمين

## ملخص

إضافة نظام مصادقة كامل مع صفحة تسجيل دخول، حماية جميع الصفحات، وصفحة إدارة مستخدمين مع التحكم بالصلاحيات.

---

## المرحلة 1: قاعدة البيانات

البنية الحالية جاهزة بالفعل:

- جدول `profiles` موجود مع trigger `handle_new_user` لإنشاء ملف شخصي تلقائيا عند التسجيل
- جدول `user_roles` موجود مع enum `app_role` (admin, manager, user)
- دوال `is_admin()` و `is_manager_or_admin()` موجودة

**المطلوب:** إضافة سياسة RLS لجدول profiles تسمح بالإدراج التلقائي (للـ trigger)، وتفعيل تأكيد البريد الإلكتروني تلقائيا حتى يتمكن المستخدمون من الدخول فورا بدون تأكيد بريد.

## المرحلة 2: إنشاء صفحة تسجيل الدخول

ملف جديد: `src/pages/Auth.tsx`

- نموذج بسيط بالبريد الإلكتروني وكلمة المرور
- تبويبان: تسجيل دخول / إنشاء حساب
- تصميم متناسق مع باقي التطبيق (RTL، عربي)
- استخدام `supabase.auth.signInWithPassword` و `supabase.auth.signUp`

## المرحلة 3: حماية الصفحات (Auth Guard)

ملف جديد: `src/components/auth/ProtectedRoute.tsx`

- يتحقق من وجود جلسة مستخدم نشطة
- يعيد توجيه المستخدم غير المسجل إلى صفحة `/auth`
- يستمع لتغيرات حالة المصادقة عبر `onAuthStateChange`

تعديل: `src/App.tsx`

- لف جميع المسارات (ما عدا `/auth`) بمكون `ProtectedRoute`
- إضافة مسار `/auth` و `/users`

## المرحلة 4: زر تسجيل الخروج

تعديل: `src/components/layout/Header.tsx`

- ربط زر "تسجيل الخروج" بـ `supabase.auth.signOut`
- عرض اسم المستخدم الحالي

## المرحلة 5: صفحة إدارة المستخدمين

ملف جديد: `src/pages/Users.tsx`

- عرض جدول بجميع المستخدمين (من جدول profiles)
- عرض صلاحية كل مستخدم (admin / manager / user)
- إمكانية تغيير الصلاحية (للمدير فقط)
- زر إضافة مستخدم جديد (يفتح نموذج تسجيل)

تعديل: `src/components/layout/Sidebar.tsx`

- إضافة رابط "إدارة المستخدمين" في القائمة الجانبية

## المرحلة 6: إنشاء أول مستخدم مدير

- عند أول تسجيل حساب، يتم إضافة دور admin تلقائيا للمستخدم الأول
- أو نستخدم edge function لإنشاء المستخدم الأول كمدير

## التفاصيل التقنية

### تدفق المصادقة

```text
المستخدم يفتح التطبيق
  -> ProtectedRoute يتحقق من الجلسة
    -> لا توجد جلسة -> توجيه إلى /auth
    -> توجد جلسة -> عرض الصفحة المطلوبة
```

### الصلاحيات


| الدور   | الصلاحيات                                                                                                                                                                |
| ------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| admin   | كل شيء + إدارة المستخدمين + حذف البيانات + الإعدادات                                                                                                                     |
| manager | العمليات اليومية + الخزينة + التقارير                                                                                                                                    |
| user    | البيع السريع + عرض المنتجات فقط + بيع للتامين و عرض حسابات الزبائن , عندما يحاول احد الدخول علي خانه ليس له صلاحية بها يطلب منه رقم السري للمدير حتي يتمكن بعرضها&nbsp; |


### المستخدم الأول

سيتم إنشاء edge function بسيطة تتحقق إذا كانت قاعدة البيانات فارغة من المستخدمين، وإذا كان كذلك تمنح أول مستخدم يسجل دور admin تلقائيا.
