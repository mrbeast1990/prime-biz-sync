

# خطة التعديلات

## 1. مشكلة تاريخ 1/1/1970 في لوحة التحكم

**السبب:** كثير من المنتجات لديها `has_expiry = true` لكن `expiry_date = null`. عند تنفيذ `new Date(null)` ينتج تاريخ 1 يناير 1970 (epoch). الفلتر يعتبرها "قاربت على الانتهاء" لأنها أقدم من 3 أشهر.

**الحل:** تعديل الفلتر في `Dashboard.tsx` و`AlertCard.tsx` ليتحقق من وجود `expiry_date` فعلياً قبل المقارنة:
```
if (!p.has_expiry || !p.expiry_date) return false;
```

## 2. تغيير "السلة" إلى "إصدار فاتورة" + إعادة ترتيب منطقة الدفع

### البيع السريع (`POS.tsx`)
- تغيير عنوان "السلة" (سطر 301) إلى **"إصدار فاتورة"**
- تغيير النصوص "السلة فارغة" إلى "لا توجد أصناف"
- إعادة تصميم منطقة الدفع السفلية (سطور 336-380):
  - وضع زر نوع البيع (نقدي/بطاقة/آجل...) في **كرت مربع صغير** على اليمين
  - بجانبه مباشرة الإجمالي بحجم **أكبر** يستغل المساحة المتبقية
  - زر التأكيد أسفلهما

### مبيعات التأمين (`InsurancePOS.tsx`)
