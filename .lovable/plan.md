

# خطة إضافة نظام بيع التأمين + نظام المشتريات

## ملخص المطلوب
1. صفحة جديدة "البيع لمستخدمين التأمين" بنفس تصميم البيع السريع، مع خزينة منفصلة وإدارة عملاء التأمين
2. صفحة مشتريات كاملة مع اختيار/إضافة مورد وإضافة أصناف مع إمكانية الانتقال لإضافة صنف جديد

---

## التفاصيل

### 1. تحديث الأنواع (Types)

إضافة في `src/types/index.ts`:
- `InsuranceCustomer`: اسم العميل، رقم البطاقة، رقم الهاتف، تاريخ البيع، الرصيد
- `InsuranceSale`: بيانات عملية البيع مع ربطها بعميل التأمين
- `PurchaseOrder`: فاتورة مشتريات مرتبطة بمورد

### 2. تحديث البيانات التجريبية

إضافة في `src/data/mockData.ts`:
- بيانات عملاء تأمين تجريبية
- بيانات مبيعات تأمين تجريبية
- بيانات مشتريات تجريبية

### 3. صفحة البيع لمستخدمين التأمين

إنشاء `src/pages/InsurancePOS.tsx`:
- نفس تصميم صفحة POS.tsx بالضبط (باركود، بحث، شبكة منتجات، سلة)
- عند الضغط على "بيع": يظهر نافذة (Dialog) لإدخال اسم العميل (مع بحث تلقائي في العملاء الموجودين أو إضافة جديد)
- بعد البيع: يتم تسجيل العملية في خزينة مبيعات التأمين المنفصلة
- حفظ العميل تلقائيا في قائمة عملاء التأمين

### 4. صفحة إدارة عملاء التأمين

إنشاء `src/pages/InsuranceCustomers.tsx`:
- جدول بجميع عملاء التأمين
- تعديل: الاسم، رقم البطاقة، رقم الهاتف
- عرض تاريخ عمليات البيع لكل عميل

### 5. صفحة المشتريات

إنشاء `src/pages/Purchases.tsx`:
- الخطوة الأولى: اختيار المورد من القائمة الموجودة، أو إضافة مورد جديد عبر نافذة سريعة
- الخطوة الثانية: إضافة الأصناف - البحث بالاسم أو الباركود
  - إذا الصنف غير موجود: زر "إضافة صنف جديد" ينقل المستخدم لصفحة المنتجات `/products` لتعريف الصنف، ثم يعود
- عرض ملخص الفاتورة مع إمكانية تعديل الكميات والأسعار
- حفظ الفاتورة وتحديث المخزون تلقائيا

### 6. نافذة إضافة مورد سريعة

إنشاء `src/components/purchases/SupplierModal.tsx`:
- نافذة بسيطة لإدخال بيانات المورد (اسم، هاتف، عنوان) بسرعة دون مغادرة صفحة المشتريات

### 7. تحديث الشريط الجانبي

تحديث `src/components/layout/Sidebar.tsx`:
- إضافة "البيع للتأمين" فوق "البيع السريع"
- إضافة "المشتريات" بعد الفواتير
- إضافة "عملاء التأمين" تحت جهات الاتصال

### 8. تحديث التوجيه

تحديث `src/App.tsx`:
- إضافة مسارات: `/insurance-pos`, `/insurance-customers`, `/purchases`

## التفاصيل التقنية

### هيكل الملفات الجديدة
```text
src/
  pages/
    InsurancePOS.tsx        -- صفحة بيع التأمين
    InsuranceCustomers.tsx  -- إدارة عملاء التأمين
    Purchases.tsx           -- صفحة المشتريات
  components/
    purchases/
      SupplierModal.tsx     -- نافذة إضافة مورد
    insurance/
      InsuranceCustomerDialog.tsx  -- نافذة اختيار/إضافة عميل تأمين
```

### تدفق المشتريات
```text
اختيار/إضافة مورد --> إضافة أصناف (بحث أو انتقال لبطاقة صنف) --> مراجعة --> حفظ
```

### تدفق بيع التأمين
```text
اختيار منتجات --> ضغط "بيع" --> إدخال اسم العميل --> تأكيد --> تسجيل في خزينة التأمين
```

### الملفات المعدلة
- `src/types/index.ts` - إضافة الأنواع الجديدة
- `src/data/mockData.ts` - إضافة بيانات تجريبية
- `src/components/layout/Sidebar.tsx` - إضافة الروابط الجديدة
- `src/App.tsx` - إضافة المسارات
