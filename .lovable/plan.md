# خطة إعادة هيكلة المنظومة وإضافة الصفحات الناقصة

## المشاكل الحالية

1. في صفحة عملاء التأمين: مكتوب "إجمالي المشتريات" والصحيح "إجمالي المبيعات"
2. روابط في الشريط الجانبي لا تعمل: جهات الاتصال، الفواتير، دفتر الحسابات، الإعدادات
3. عملاء التأمين منفصلين عن الحسابات - يجب دمجهم في صفحة حسابات موحدة

---

## الصفحات الجديدة والتعديلات

### 1. إصلاح صفحة عملاء التأمين

- تغيير "إجمالي المشتريات" الى "إجمالي المبيعات"

### 2. صفحة الحسابات الموحدة (`/accounts`)

صفحة واحدة بثلاث تبويبات (Tabs):

- **الزبائن**: قائمة العملاء العاديين مع الرصيد
- **الموردين**: قائمة الموردين مع الرصيد
- **عملاء التأمين**: قائمة عملاء التأمين مع بيانات البطاقة

عند النقر على أي شخص (زبون/مورد/عميل تأمين):

- يفتح Dialog يعرض تفاصيل الحركة: جميع الفواتير والمبيعات المرتبطة به
- عرض الرصيد الحالي والمبلغ المدفوع والمتبقي
- إمكانية تعديل البيانات (الاسم، الهاتف، العنوان، رقم البطاقة لعملاء التأمين)

### 3. صفحة الخزينة (`/treasury`)

- عرض إجمالي الرصيد النقدي الحالي
- قسم خزينة المبيعات العادية (النقدية والبطاقة)
- قسم خزينة مبيعات التأمين (منفصلة)
- قسم المصروفات (المشتريات)
- سجل الحركات المالية (دخول/خروج) مع التاريخ والمبلغ والمرجع
- إمكانية إضافة حركة يدوية (إيداع/سحب)

### 4. صفحة التقارير (`/reports`)

- تقرير المبيعات اليومية/الأسبوعية/الشهرية مع رسم بياني
- تقرير المشتريات
- تقرير الأرباح (المبيعات - تكلفة البضاعة)
- تقرير مبيعات التأمين منفصل
- تقرير المنتجات الأكثر مبيعا
- تقرير حركة المخزون
- إمكانية فلترة التقارير بالتاريخ

### 5. صفحة الإعدادات (`/settings`)

- اسم الصيدلية وبيانات التواصل
- &nbsp;
- إعدادات الطباعة (حجم الفاتورة)
- &nbsp;

### 6. تحديث لوحة التحكم

- إضافة ملخص الخزينة (رصيد اليوم)
-  إضافة ملخص مبيعات التأمين اليومية بعدد العملاء ليس بالقيمة
- تحسين بطاقات الإحصائيات

## التفاصيل التقنية

```text

```

### الملفات المعدلة

- `src/pages/InsuranceCustomers.tsx` -- تصحيح "المشتريات" الى "المبيعات"
- `src/components/layout/Sidebar.tsx` -- إعادة هيكلة القائمة
